Folder · split

Two top-level branches under trackingtogether_/: owned_ and clients_. The split is not stylistic — it enforces IP boundary, equity boundary, access-rule boundary at the filesystem level.

trackingtogether_/
  ├── owned_/                  # AV-owned ventures, equity + IP retained
  │   ├── net_meter_/
  │   ├── pfc_farms_/
  │   ├── ixl_/                # paired with Net Meter
  │   └── _holdings_-private_/
  └── clients_/                # TT runs automations FOR them, IP to client
      ├── acme_/               # + other auto-vendor list
      ├── culligan_/
      └── ...

Why split is non-negotiable: Flat folder blurs the IP boundary. Eli's per-job context resolution becomes ambiguous (own project vs client project = different deliverable rules, different access posture, different revenue treatment). The split makes IP / access / equity legible from the path alone.
