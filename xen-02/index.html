// System Architecture

How XEN
is built

XEN is not a monolith. It is a fleet architecture — dozens of specialized daemons, each with a single responsibility, coordinated through a universal hub that never sleeps.

The design principle: no single point of failure, no single thread of execution. Every daemon has a watchdog. Every watchdog has a stale-detector. The system heals itself before you notice the wound.

  • Universal MCP hub on port 23375
  • 50+ MCP servers, hot-pluggable
  • Canonical tree on M4 NVMe only
  • No sync I/O on any hot path
  • Write-tmp-then-rename on all state
  • Every daemon has a heartbeat watchdog
  • Runtime state under ~/.xen/runtime
  • Vault on M4, 83+ memory entries
