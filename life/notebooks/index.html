How to centralize on xlrdtech

the exact steps · share = a Google permission action
  1. Decide the model. "One spot" = either (a) share each non-xlrdtech notebook TO xlrdtech@gmail.com so it appears under one login, or (b) keep the wired-profile model where the CLI/lens reaches all accounts. Sharing gives a single Google login; profiles give a single CLI. qi picks.
  2. For the 28 hero notebooks (share path): in a browser logged in as hero@l7sinc.com, open each notebook → click Share (top-right) → add xlrdtech@gmail.com as Editor → Send. This is an owner-permission action — only qi (or qi-authorized browser auth) may perform it. East does not change sharing permissions.
  3. Authenticate the 3 pending accounts (gxdcult, god.galleryllc, judahjohanson): cookie-harvest each Google session via the Comet daemon (comet-nblm-harvest), or notebooklm login per profile. Then they become enumerable.
  4. Re-run the inventory across all 5 profiles to get the true total and the full list of owned-elsewhere notebooks to share.
  5. Batch-share what's left. Repeat step 2 for every owned notebook in each newly-authenticated account, adding xlrdtech@gmail.com as Editor.

NotebookLM has no API/CLI command to add a share recipient — sharing is dashboard-only and owner-gated. The safe, fully-automatable part is enumeration (done here for the 2 wired accounts) and auth-harvest; the share clicks themselves need qi or qi-authorized browser auth.
