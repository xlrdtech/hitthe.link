Stripe Products, Pricing & Checkout

The L7S revenue rail — every offer becomes a Stripe product with a checkout link, and a paid checkout auto-starts onboarding.
What this builds Each L7S offer maps to a Stripe product + price + checkout link. A successful payment fires the flow into onboarding (Yes→Pay→Sign→Upload), and recurring / custom work bills through Stripe Invoicing. This is the setup blueprint — create each item in Stripe, paste the live link into the matrix, ship.
1.

The Revenue Flow

One path from offer to paid-and-onboarding. Mirrors the Onboarding system (Yes→Pay→Sign→Upload).
Offer chosen→ Stripe Checkout link→ Payment success→ Log client in Asana (Client Onboarding)→ Sign agreement→ Secure upload
  • Trigger on paid, not on promise.The onboarding record is created only when Stripe reports the payment succeeded (webhook checkout.session.completed) — no chasing.
  • One link per offer.Every package and add-on is its own Stripe Payment Link, shareable in SMS / WhatsApp / the site.
  • Onboarding handoff.Success page → /l7s/onboarding/ (sign) → /l7s/upload/ (secure intake).

3.

v1 Offer Checkout East · ship first

The single live offer to stand up immediately so money can move this week.
  • Product: L7S — v1 Offer (the current lead package East is selling).
  • Price: Set to the v1 amount (confirm against PriceHero) and create a single Stripe Payment Link.
  • Link name: “Stripe — v1 Offer Checkout” — this is the link dropped into the thread/SMS when a prospect says yes.
  • On success: Route to onboarding + log the client. No delays — paid is the green light.
4.

Invoicing

  • Stripe Invoicing for custom + retainer.Anything not a fixed package (scoped builds, milestones, retainers) bills via a Stripe invoice with the L7S brand, due-date, and auto-reminders.
  • Keys + tracking on the portal.Wire the Stripe keys so payments and invoicing track to the portal — one ledger for paid / pending / overdue.
  • Pricing source of truth.All amounts derive from the offer ladder — keep PriceHero and Stripe in sync so quotes match checkout.
5.

Stripe Setup Checklist

  • Create a Product for each row in the matrix.
  • Add a Price (one-time or recurring) to each product per the ladder.
  • Generate a Payment Link per price; paste the live URL into the matrix.
  • Set each link's success URL → /l7s/onboarding/.
  • Add a webhook on checkout.session.completed → create the Asana “Client Onboarding” record.
  • Enable Stripe Invoicing for custom / retainer work.
  • Confirm amounts against PriceHero; ship.
