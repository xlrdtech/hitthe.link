How this is now mitigated — what stops the next lie

01 · UPSTREAM BRAKEHOOK
truth-guard · fail-open
Every agent turn is scanned
A Stop + PostToolUse hook flags any unverified “done / shipped / all complete” with no receipt in the same breath, and tells the agent to downgrade to shipped-unverified or pending. It only ever flags — never blesses, never blocks a tool.
02 · PROOF, NOT WORDSCLI
xen-verify · fail-closed
Turns “shipped” into evidence
Pulls the real bytes: url wants HTTP 200 + real size + no template tokens; apk demands >1MB + a real manifest — the exact gate that caught tonight’s 12.7KB stub. Unprovable means FALSE.
