How it works

I

You speak or type

Voice injection reaches Xen sub-400ms. Text lands the same moment. Either path triggers parallel dispatch — no queue, no poll loop.

II

Goals fan out immediately

Every non-trivial task spawns a Parallel Goal Swarm. Each sub-task runs as its own background fork. Nothing blocks the main thread.

III

Memory is consulted first

Before any multi-step plan, the Bastra vault surfaces relevant context. Decisions made yesterday are available today without re-explaining.

IV

Output reaches you three ways

Voice reply, SMS to both lines, and a live visual surface — simultaneously, in the same batch. Silence between commands is a violation.

V

Work continues after you stop

Every wakeup converts to a concrete advance on the next open goal. There is no idle state. No task left pending if it can be moved forward now.
