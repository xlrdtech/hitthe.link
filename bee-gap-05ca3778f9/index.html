Bee Capability Gap Report

01

What we actually use today

Only two live features. Everything below section 3 is available but unused.

LIVE Ambient capture

bee stream --json

The only live capture path. bee-ambient-bridge.py (~line 239) spawns it with no --types and no webhook, dedups the double-pulse, POSTs each utterance to omnimind, and respawns on a 3s backoff.

LIVE Auth keepalive

bee status

A 60s keepalive tick (xen-bee-mcp-keepalive.sh) that holds the CLI warm and authed so the stream never drops its session.

IDLE MCP server

bee mcp serve

Registered in ~/.claude.json exposing 26 mcp__bee__* tools — but nothing in the stack calls them. Available, authed, and completely unused.

02

Top 5 unused wins

Build in this order. Each closes a real gap in the live loop.

1

Cut ambient latency at the source qi's live pain

bee stream --webhook-endpoint <url> --webhook-body '{"message":"{{message}}"}' makes Bee POST each utterance directly to an omnimind endpoint, deleting the Python stdout-reader middleman and the respawn backoff. Alternatively bee proxy [--port N | --socket] is a local HTTP or unix-socket proxy to the Bee API (socket = lowest overhead). Either directly cuts the ambient-inject hop.

Reality check: the delay is mostly Bee's own upstream transcription + phone-sync. Our bridge is already line-buffered and threaded-POST, so it's near-zero on our side. Webhook removes the small local hop; Bee-server latency remains.
2

Widen capture with typed / agent streams

The bridge hardcodes default stream types. bee stream --types all widens capture to utterances + conversation updates + summaries; --agent gives a purpose-built agent-mode stream.

Caution: earlier this session --types all flooded the bridge's summarize() which drops update-conversation events, so nothing landed — it was reverted. Re-trying --types all requires updating summarize() to handle the extra event types.
3

Auto-surface spoken action-items

bee todos suggestions + mcp__bee__bee_get_todo_suggestions — Bee auto-derives action items from qi's speech and we ingest zero of them. Wire this read-only into omnimind so spoken action-items auto-surface into the goal system.

4

Nightly daily-summary recall

bee today --context + bee_get_daily_summary / bee_list_daily_summaries — one call returns the daily summary plus todos, notes, and captured conversations. Pull nightly and write to the M4 vault for Akashic recall.

5

Mirror all Bee data to markdown

bee sync [--output <dir>] [--only ...] [--full] mirrors all Bee data to markdown. Point --output at the M4 vault for offline recall and xen-search grep over the full Bee history.
