The five domains — drill in weight order

1 · Agentic architecture & orchestration 27%

  • Claude is stateless — your code runs the loop: call → inspect → execute → append → call.
  • Spot the broken step (drop append → the model forgets the tool result).
  • Manage context budget by spawning sub-agents via a task tool.
  • Sequential (step-dependent) vs Adaptive (open-ended) decomposition.

2 · Claude Code configuration & workflows 20%

  • claude.md hierarchy: user-level (~/.claude/claude.md, not versioned) → project-root (team) → directory-level (subtree).
  • Custom slash commands via YAML front-matter restrictions.
  • Headless CI/CD runs with the -p flag.

3 · Prompt engineering & structured output 20%

  • Replace vague adjectives with categorical, mechanically-checkable rules.
  • JSON via tool shape + tool_choice, plus a validate-and-retry loop.
  • Message Batches API = 50% per-token cost for offline work.

4 · Tool design & MCP integration 18%

  • Write tool descriptions like API docs to stop random tool selection.
  • Failures return structured error JSON: is_error, category, retryable, retry_after_ms.
  • Transport: stdio for local processes, SSE for remote hosts.

5 · Context management & reliability 15%

  • The “lost in the middle” effect — anchor durable facts in a case block at the very end, not a summary.
  • Prompt caching: ~5-min TTL, ~1/10th input price.
  • Escalation: flag low confidence / ambiguity and hand off to a human.

The five distractor traps wrong-answer DNA

1

Vague-adjective trap — adding “careful / thorough” instead of numbered categorical rules.

2

Summarization trap — summarizing a long chat (lossy, drops numbers & IDs) instead of extracting into a case block.

3

Blanket-retry trap — re-running every sub-agent after one fails (duplicate rows / emails) instead of a targeted retry.

4

User-claude.md trap — putting team-shared rules in the user-level file, which never reaches version control.

5

SSE-on-localhost trap — SSE transport for a co-located client instead of stdio.

Four-week plan preporato.com

Week 1

Domain 1. Read the guide; build an agent loop + a hub-and-spoke orchestrator; take practice exam 1 as a baseline.

Week 2

Domains 2 & 3. Guides + cheat sheets; a Claude Code commands project and a prompt/eval project; practice exams 2 & 3.

Week 3

Domain 4. MCP guide; build an end-to-end MCP server handling structured errors & secrets; practice exam 5.

Week 4

Domain 5. Crash-recovery / compaction projects; sit practice exams 4 & 6 back-to-back under the full 2-hour timer.
