The ship pipeline was delivering into a void

A 6-dimension audit of autoship: what it promises (canon) vs what it actually does. The headline finding is blunt — qi hadn't received a single autoship delivery since June 30, because the delivery leg pointed at a platform that no longer exists. That leg is now repaired and verified live.

6 gaps found 2 critical delivery leg REPAIRED 2026-07-02

01Top line

qi had not received a single autoship delivery since 2026-06-30.

Step 7 sent to a dead Beeper chatID (mcp__beeper__send_message → a homeserver that no longer exists), and the "corrected" Matrix path was itself broken three ways: the helper read matrix_room_ids.json which has no ship_feed key, it omitted the Cloudflare-mandatory browser User-Agent, and it had no retry/fallback/durable log — so a wedged Synapse would lose the URL forever.

Every leg of delivery was broken at once. From qi's phone, the pipeline was dead — and 218 commits of heartbeat noise since 6/30 confirmed nothing real reached him. Meanwhile throughput ran ~7 real ships in 24h against a 48/day floor: ~85% under.

02Ranked gaps

Ordered most-to-least damaging. Severity is tied to qi's economics: a live page he never sees is not shipped, and unseen work burns the $200/hr green-light debt.

Delivery leg dead — 4 compounding failuresCritical

Step 7 targeted a retired Beeper tool/room; even the canon-blessed Matrix path failed because matrix_send.py read the wrong room-map file, omitted the WAF User-Agent, and had no durable log — so a live URL vanished with no trace when Synapse wedged.

Fix · SMALL · DONE — repointed matrix_send.py to matrix_chat_map.json, added the browser UA to both request paths, made the lookup skip metadata keys. Delivery to !cVElhceMpRmZVQkNnU verified with a real event_id. SKILL.md step 7 + all Beeper refs rewritten to Matrix.

No MAIN-session guardCritical → High

"Steps 6–7 must run from MAIN" was prose with zero enforcement, so a subagent silently opened/delivered nothing — the root of the historical >50% open-failure rate.

Fix · TRIVIAL · DONE — added a hard guard: if ($env:XEN_NESTED) { throw } at the top of the open/deliver block. A silent miss becomes a visible handoff back to MAIN.

Stale-cache / content-blind live gateHigh

The gate (200 + >500B + no {{) proves a page is live, never that the new build is — a re-ship over a warm cache passes showing yesterday's content, and the 150s ceiling is below GH Pages' own 3-min worst case.

Fix · SMALL · queued — stamp <!--build:<sha>-->, poll ?v=<sha> to force origin, require the sha marker present, broaden the leak set, raise ceiling to 210s.

48/day floor unmeasured & unwiredHigh

The floor is a mandate with zero implementation — nothing counts real ships (the only ledger died 2026-05-19), nothing excludes noise commits, and the ambient auto-fire loop is canon prose with no hook. Result: ~7/48, invisibly.

Fix · SMALL+MED · queued — a ship-count-today counter (git log minus noise regex, prints N/48) + surface it as step 9 + fire autogoal when under-floor. Auto-fire cadence needs qi's OK (usage-burn is live).

No breath-to-live economicsMedium

The $111 / $3.33-per-min / 33:33 break-even canon is entirely absent — a 3-min ship and a 50-min ship report identically, so qi can't see when a unit runs into debt.

Fix · SMALL · queued — stamp breath-time at qi's utterance, surface elapsed on ship, flag >33:33 as debt, carry the number into the delivery message.

Ambient auto-fire & idempotency unbuiltMedium

Canon says autoship auto-fires per inject with same-topic-steer / new-topic-fork, but nothing fires until qi says the word, and there's no goal-ledger for same-vs-new dedup — so forgotten injects never ship and re-runs risk double-ship.

Fix · TRIVIAL+SMALL · queued — route buildable injects through /post by default + a flat-file active-ships.jsonl dedup store. (Unbounded per-keystroke swarms stay aspirational — they'd burn usage.)

03autoship is the tail of the auto_ chain

qi's note (2026-07-02): autoship requires most/all of the auto skills. It is NOT a standalone git pipeline — it's the SHIP step of the continuous loop, and the rest of the auto_ family feeds it.

SkillRole in the chain
btw / postIngest — the gateless close-the-loop that fuses absorb + autogoal + autoship into one motion. When qi drops context, /post is the front door, autoship is its tail.
autogoalBPGS — what qi says becomes a goal, goals fan out into a swarm; autoship is how each swarm goal reaches LIVE. Under-floor days fire MORE autogoal to clear 48/day.
forkEach distinct thing forks its own parallel ship; same-topic re-raise steers the in-flight ship (idempotent), never double-ships.
ambient loopEvery buildable inject routes through btw → autogoal → fork → autoship → repeat by default — not waiting for the literal word "ship".

Now documented in the SKILL.md: if you invoke autoship in isolation for anything bigger than a one-file edit, you're under-running it — pull in autogoal/post so the ship is goal-tracked, floor-counted, and idempotent.
