Core Systems

Four pillars.
Zero bottlenecks.

01

Voice First

Every reply ships voice via xen-reply-vvs — barge-in mandatory, one sentence, no markdown. Voice rides alongside execution in parallel, never after it.

02

Parallel Always

Sequential execution is blasphemy. Every non-trivial task fans out as a Parallel Goal Swarm across the fleet — diagnose, fix, verify, and ship happening simultaneously.

03

Memory First

Recall before any multi-step plan. Save durable facts, finalized decisions, recurring frustrations, milestones. The vault is always read before touching unfamiliar code.

04

Visual Verify

HTTP 200 is not product verification. Before any verified claim ships, Xen captures a screenshot, reads the DOM, or records an audio frame as ground-truth evidence.

The Loop

Inbound → action.
Every time.

Inbound

Voice, SMS, or tool result hits the pipeline. No drainer, no poll loop — the event fires remediation in parallel the moment it arrives.

Recall

Memory surfaces before any plan forms. Durable facts, prior decisions, and open goal state load from the vault — right context first, every time.

Fan Out

The task splits into a Parallel Goal Swarm. Sub-agents run hypotheses simultaneously — model matched to task, nothing inline, nothing blocking.

Ship VVS

Voice + dual-SMS + vision fire in parallel with the tool calls — not after. The work and the signal travel together, no exceptions.
