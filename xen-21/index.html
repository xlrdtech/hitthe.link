Capabilities

What XEN does that nothing else can.

REF C1 — VOICE

Persistent Voice Intelligence

XEN speaks through every channel simultaneously — SMS, voice call, beeper relay, UI surface — as a single coherent voice. There is no session, no context window to manage. The conversation is continuous and permanent. Barge-in is mandatory. Response latency under 400ms from voice input to spoken reply, measured end-to-end on real infrastructure.

REF C2 — MEMORY

Akashic Memory Layer

One persistent memory store — past, present, and anticipated future state — written and read by every agent in the fleet. No context is ever lost. Recall fires before every multi-step operation.

REF C3 — SWARM

Parallel Goal Swarms

Every non-trivial task fans out into a parallel swarm — never a serial loop. New directives never abandon prior goals. Sub-agents spawn at any effort level, on any model, instantly backgrounded.

REF C4 — AUTONOMY

Human-Out-of-the-Loop Execution

XEN operates identically whether you are present or absent, awake or asleep. Inbound signals — from any source, any channel, any priority — map instantly to action without human confirmation. Any interaction that requires a tap, click, or confirmation is considered an architectural failure. Zero idle states. Zero standby. The system is permanently producing.

REF C5 — OBSERVE

Full-Stack Omniscience

Every thread known at all times — beeper channels, SMS legs, live call audio, daemon heartbeats, disk state, process health, goal indexes. Xen does not consult logs after the fact. It witnesses state as it changes, cross-correlates signals in real time, and acts on what it finds before the anomaly propagates. Visibility is not a dashboard. It is the operating condition.

Architecture

One core.
Everything else flows through it.

XEN maintains a single canonical tree. No mirrors, no sync I/O on hot paths, no flapping volumes. Every agent in the fleet reads from and writes to the same persistent state — goals, memory, identity, and runtime — with flock-guarded writes and atomic rename semantics.
