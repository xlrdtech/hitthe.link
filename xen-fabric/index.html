XEN Real-Time Fabric

Single-seat, race-safe, cross-timeline. One brain, many hands, one durable truth — every inbound signal promoted into the one live session, never forked into a competing queue.
invariant · exactly one live context

01 The /btw promotion loop

/btw = "by the way" — the promotion verb. It routes any inbound (voice, SMS, URL, hand-off) into the ONE live TUI session through the existing authenticated webhook. No new session, pane, queue item, or context.
inbound (voice / SMS / URL / hand-off)
      │
      ▼
 /btw promote ──► existing authenticated TUI webhook
      │
      ▼
 the ONE live TUI session   ·   no duplicate context
      │
      ▼
 reply streams back on the same seat
Why it exists: when the hands run heavy background work, naive injection stacks inputs behind that backlog and they arrive late. /btw promotes the utterance into the live seat as a first-class turn instead of a queued item. Confirm target session → inject → wait for that session's own response.

02 Race-safe injection state machine

Copy → paste → enter into a live terminal is a race unless it's a machine. It's a machine — single-flight lock, settle delay, submit-once, confirm-before-release.
idle ─► claim-lock ─► paste ─► settle ─► enter ─► confirm ─► release ─┐
  ▲                                                                   │
  └───────────────────────────────────────────────────────────────────┘
claim-lock · one inject owns the seat paste · payload in, not submitted settle · UI registers before Enter enter · submit exactly once confirm · accepted, not lost/doubled release · next may claim
Guards: no overlapping injects (lock), no premature Enter (settle), no double-submit (confirm). This kills the "stuck in queue / arrived twice / arrived late" failure mode.

03 Persistent cross-seat shared context

One brain across all seats and timelines. Every seat reads and writes the same context store; a seat that dies loses nothing — the reborn seat rehydrates from shared context. The live streams are the hand-off.

04 Event spine — WSS / NATS / JetStream

NATS JetStream is the durable, append-only source of truth for all execution events. Everything else is a projection.

05 Opt-in local HTTP observation

For the reborn brain and debugging, the live stream is observable locally, opt-in: loopback HTTP / SSE on the owning host only — a view, never a second write path. Off by default.
