01 The store

This folder is the memory — and an Obsidian vault.

  • Location — the Claude project memory dir on nitro (registered Obsidian vault with graph + backlinks).
  • One note = one fact. Frontmatter carries name · description · type · tags; [[wikilinks]] connect facts.
  • Types (color-grouped in Graph View): user · feedback · project · reference · system.
  • MEMORY.md — the index (one line per note) and the always-auto-loaded session context. The graph hub.

02 Harness wiring

Why it works where a bare vector store produced ~nothing: hooks remove the "remember to save" burden.

LifecycleHookAction
SessionStartensure-inject-daemondeterministic context inject layer
UserPromptSubmitmemory-capture · INLLM-free append every prompt → NDJSON
PostToolUsememory-capture · TOOLappend tool events → NDJSON
Stopmemory-autosave + capture · OUTgit commit + push vault; capture final output

Capture log self-caps at 25 MB (→ last 2000 lines). Context is never lost even when no note is written — a detached distiller can later propose notes from the log.

03 Durability — 4 safety nets

  • git at the memory dir — Stop-hook auto-commits + pushes every session.
  • Offsite DR — private GitHub repo (SSH, main), auto-pushed by the hook.
  • Syncthing peer to Mac-Xen — .stignore excludes .git to avoid git↔Syncthing collision. re-launch if sync needed.
  • Integrity — frontmatter validation + dead/orphan-link check.

04 Obsidian — viewing

  • Installer access-violates headless — working path is to extract the app (7-Zip the setup → app-64.7z → run Obsidian.exe --disable-gpu).
  • Browser fallback — a generator emits a self-contained interactive force-graph. Local-only — never published (the notes hold PII + secret names).

05 NotebookLM

A research feeder — not the memory store.

  • Wired via the yt-pipeline skill + notebooklm-mcp (persistent auth).
  • Flow: YouTube search → NotebookLM notebook → analysis → saves a markdown report that lands in Obsidian.
  • Upstream of the vault — produces research notes, does not hold canonical memory.

⚠ Open discrepancy

yt-pipeline writes research to ~/Documents/obsidian/, while the canonical memory vault is the Claude project dir. Two Obsidian locations — decide whether research should land in the canonical vault.

06 Known-open (not yet built)

  • Detached distiller — scheduled pass that reads the NDJSON and proposes candidate notes into _candidates/ for review (must not auto-canonize → belief-poisoning guard).
  • SessionStart CONTEXT.md inject — hard-capped state briefing (today MEMORY.md auto-loads, covering most of it).
  • Freshness heartbeat — alert if capture goes stale.

07 Mac counterpart

Mac-Xen uses bastra-recall (an opaque MCP vector store). Verdict: demote it to an optional secondary index pointing at the markdown — the vault is the source of truth, never the vector DB.
