Android-MCP.

A bridge between LLM/VLM agents and a real Android device. No computer-vision model, no pre-programmed scripts — the agent reads the on-screen UI hierarchy through the Accessibility API and drives it over ADB, tool call by tool call.

View on GitHub ↗ See it drive a phone ↓

Exposed tools

tap(x, y) long_press(x, y) swipe(x1, y1, x2, y2) drag(x1, y1, x2, y2) type_text(text) press_button(key) screenshot() get_ui_hierarchy() get_notifications() shell(command)

Ten tools cover the whole interaction surface — everything a human thumb (and a `adb shell`) can do to a device.

How it connects. The server talks to a device over ADB — USB, WiFi, or auto-detected — and resolves the connection lazily on first tool call, so it starts cleanly even with no device attached yet.

Every action reads back through the Accessibility API's UI tree rather than a screenshot + vision model, so the agent gets exact element bounds, text, and clickable state instead of guessing pixel coordinates.

Requirements. Python 3.13+, the adb tool on PATH, and Android 10+ (device or emulator). Typical loop latency is 2–4s per action — the ceiling is ADB round-trips, not the model.

Install

uvx
uv
opencode
// claude_desktop_config.json
{
  "mcpServers": {
    "android-mcp": {
      "command": "uvx",
      "args": ["android-mcp-server"]
    }
  }
}
git clone https://github.com/CursorTouch/Android-MCP
cd Android-MCP
uv sync
uv run android-mcp-server --device auto
// opencode.json
{
  "mcp": {
    "android": {
      "type": "local",
      "command": ["uvx", "android-mcp-server"]
    }
  }
}
